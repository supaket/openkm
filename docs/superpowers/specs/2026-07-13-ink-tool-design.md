# Ink Tool — ปากกา Apple Pencil สำหรับหน้าติวหนังสือ (law-legal + utcc)

**วันที่:** 2026-07-13 · **สถานะ:** รอผู้ใช้อนุมัติสเปกก่อนลงมือ

## เป้าหมาย

ให้ทุกหน้าติวใน `books/law-legal/` (18 หน้า) และ `books/utcc/` (21 หน้า) เขียน/ไฮไลต์ด้วย
Apple Pencil, เมาส์ หรือนิ้วได้ มีแถบเครื่องมือหุบ-กางได้ บันทึกผลเป็นภาพ PNG ลงเครื่องได้
และเส้นที่จดไว้คงอยู่เมื่อกลับมาเปิดหน้าเดิม

## สถาปัตยกรรม

- ไฟล์เดียว: **`assets/ink-tool.js`** (vanilla JS, ไม่มี dependency ยกเว้น html2canvas โหลด
  แบบ lazy ตอนกดปุ่มบันทึกภาพเท่านั้น)
- แต่ละหน้าเพิ่ม 1 บรรทัดก่อน `</body>`:
  `<script src="../../assets/ink-tool.js" defer></script>` (ปรับ `../` ตามความลึกโฟลเดอร์)
- ทำงานได้ทั้งหน้า HTML ธรรมดาและหน้า React/antd เพราะไม่แตะ DOM ของเนื้อหา —
  แค่วาง overlay ทับ

### องค์ประกอบ

| ส่วน | หน้าที่ |
|---|---|
| Ink canvas | `position:absolute` ทาบเต็มความสูงเอกสาร → **เส้นติดกับเนื้อหา เลื่อนตามหน้า** |
| Toolbar (FAB) | ปุ่ม ✎ ลอยมุมขวาล่าง (`position:fixed`) กดกาง/หุบ · Esc เพื่อหุบ |
| Stroke store | เก็บเส้นเป็น vector: `{tool, color, points:[[x,y,pressure],…]}` พิกัดอิงเอกสาร |
| Persistence | บันทึก strokes ลง `localStorage` key = `ink:<pathname>` (debounce หลังจบเส้น) |
| Capture | html2canvas (CDN, โหลดเมื่อใช้) วาด DOM + ink canvas → PNG download |

## เครื่องมือบนแถบ

1. ปากกา 3 สี (ทอง `#c9a227` / แดง `#e0433f` / น้ำเงิน `#2563eb`) — ความหนาแปรตามแรงกด Pencil
2. ไฮไลต์โปร่งแสงสีเหลือง (เส้นหนา, `multiply`-style opacity)
3. ยางลบ (ลบทีละเส้นที่แตะโดน — ลบจาก stroke store แล้ววาดใหม่ ไม่ใช่ลบ pixel)
4. ↩️ Undo ทีละเส้น
5. 🗑️ ล้างทั้งหน้า (ยืนยันก่อน)
6. ✍️/✋ โหมดกันฝ่ามือ: ✍️ = รับเฉพาะ `pointerType==='pen'` (นิ้วเลื่อนหน้าปกติ) —
   เปิดอัตโนมัติเมื่อตรวจพบ Pencil ครั้งแรก · ✋ = นิ้ว/เมาส์เขียนได้
7. 📸 บันทึกภาพ **2 ปุ่ม**: "เฉพาะที่เห็นบนจอ" และ "ทั้งเอกสาร"

## พฤติกรรมสำคัญ

- **การเขียน:** Pointer Events + `getCoalescedEvents` (เส้นลื่นบน 120Hz) + `setPointerCapture`
  ขณะโหมดเขียนเปิด canvas เป็น `touch-action:none` เฉพาะเมื่อ pointer เป็นชนิดที่เขียนได้
- **เลื่อนหน้า:** โหมด ✍️ นิ้วเลื่อนได้เสมอแม้เครื่องมือกางอยู่ — อ่าน+จดต่อเนื่อง
- **Resize/หมุนจอ:** คำนวณตำแหน่งเส้นเป็นสัดส่วนของความกว้างเอกสาร (x เก็บเป็น fraction,
  y เก็บเป็น px คูณ scale) → หมุนจอแล้ววาดใหม่ เส้นยังอยู่ตรงเนื้อหาเดิมโดยประมาณ
- **DPR/หน่วยความจำ:** canvas ใช้ `devicePixelRatio` แต่ cap ที่ 2 และ cap ความสูง canvas
  ที่ ~16,000px (พอสำหรับทุกหน้าปัจจุบัน; ถ้าเกิน แจ้งเตือนและตัดที่เพดาน)
- **บันทึกภาพ "ทั้งเอกสาร":** อาจใช้เวลา 3–10 วิบนหน้า/เครื่องช้า — แสดงสถานะ "กำลังสร้างภาพ…"
  · ถ้า html2canvas โหลดไม่ได้ (ออฟไลน์) → fallback ดาวน์โหลดเฉพาะลายเส้นพื้นขาว
- **ชื่อไฟล์ภาพ:** `<ชื่อไฟล์หน้า>-notes-<YYYYMMDD-HHmm>.png`
- **localStorage เต็ม/ใกล้เพดาน:** แจ้งเตือนและเสนอให้ล้างหน้าเก่า (เก็บ ~5MB/origin)
- **ไม่กระทบการพิมพ์:** `@media print` ซ่อน toolbar (แต่เส้นจะติดไปด้วยถ้าพิมพ์ทั้งหน้า —
  ถือเป็น feature)

## การติดตั้ง (rollout)

1. สร้าง `assets/ink-tool.js`
2. สคริปต์ inject `<script>` 1 บรรทัดเข้า 39 ไฟล์ (18 law-legal + 21 utcc) คำนวณ `../` อัตโนมัติ
3. ทดสอบด้วย Playwright: วาดผ่าน PointerEvent (pen/mouse), ตรวจ persistence (reload แล้วเส้นคืน),
   ตรวจปุ่มบันทึกภาพได้ไฟล์จริง, ตรวจหน้า React (GA510) และหน้าธรรมดา (law1101-study-guide)
4. commit + push · ไม่ต้องแก้ TOC (ไม่มีไฟล์หน้าใหม่)
